# 工程化 🕳️

从一个朴素问题出发——**"写代码已经能跑了，为什么还要工程化？"**——一路追到模块边界、自动化、质量门禁、发布回滚和反馈闭环。

## 推荐阅读顺序

| # | 篇 | 一句话 |
|---|---|---|
| 01 | [工程化是什么](01-what-is-engineering.md) | 工程化不是把流程搞复杂，而是把个人手工成功，变成团队可重复、可验证、可恢复的稳定交付 |

## 母题

本洞继续印证 [../patterns.md](../patterns.md) 里的"分层 + 封装"和"在更弱的底座上补出更强的抽象"：人的记忆、手工操作和个人经验都不可靠，工程化用约定、工具、自动化和反馈系统，把不稳定的个人行为补成稳定的团队能力。
